# 求职智能体 — 项目演示视频录制流程

**演示平台**：AutoClaw（小龙虾）  
**LLM 后端**：GLM-4-Plus  
**演示模式**：全程自然语言交互，由 GLM 意图识别自动路由至对应 Pipeline

---

## 录制前准备

| 准备项 | 操作 |
|:--|:--|
| 清空输出 | 删除 `Job_Hunting_Copilot_Skill/output/` 下所有文件 |
| 安装 Skill 包 | 将 `Job_Hunting_Copilot_Skill/` 整体导入 AutoClaw |
| 加载配置 | 确认 `agent_config.json` 中 14 个 Skill 全部注册成功 |
| 网络环境 | 确保可正常访问招聘平台（场景三需要） |
| 录屏设置 | 终端字体 ≥18pt，深色主题，关闭系统通知 |
| 素材准备 | 提前复制好经历草稿和 JD 文本（见各场景） |

---

## 开场（约 30 秒）

**画面**：AutoClaw 主界面，展示已加载的 Skill 列表

**旁白**：

> 这是求职智能体，基于 AutoClaw 框架 + GLM-4-Plus 构建。
> 采用路径 B 原生 Skill 模式，将求职能力封装为 14 个原子化 Skill，
> 用户只需一句自然语言，GLM 自动识别意图并编排 Pipeline 执行。
> 下面演示三个核心场景。

## 场景一：个性化简历生成（约 3 分钟）

**触发 Pipeline**：`resume_generation`  
**Pipeline 链路**：JD 分析 → 经历萃取 → STAR 打磨 → Word 导出

### Step 1 — 输入自然语言指令

在 AutoClaw 对话框中输入：

```
帮我生成一份针对管培生岗位的简历
```

### Step 2 — GLM 意图识别

GLM-4-Plus 识别意图，命中关键词「生成简历」「管培生」，自动路由至 `resume_generation` Pipeline。

**画面关注点**：日志区展示意图匹配过程

### Step 3 — 上传经历草稿

当 AutoClaw 提示输入经历时，粘贴以下内容：

```
大二加入了学生会文艺部，做了一年干事，期间办了迎新晚会，拉了2000块钱赞助，还写了几篇推文。

大三暑假去了一家互联网公司实习，主要工作是回复用户消息，建了几个微信群，整理过各种表格。

大四参加了大创项目，我们组做了一个校园互助小程序，我主要负责产品方向的需求整理和用户调研。
```

### Step 4 — 输入目标岗位 JD

粘贴 JD（或由 GLM 自动生成默认 JD）：

```
管培生岗位要求：本科及以上学历，专业不限。具备较强的沟通协调能力，学习能力和抗压能力。有学生干部经验优先。熟练使用 Excel、PPT 等办公软件。具有跨部门协作经验，善于解决问题，有领导力潜质。每周5天，实习3个月以上。
```

### Step 5 — Pipeline 自动执行

画面展示 4 个 Skill 依次执行的日志：

```
[Step 1/4] > JD 分析与关键词提取
  → 提取硬技能: Excel, PPT, 数据分析
  → 提取软素质: 沟通协调, 领导力, 抗压能力

[Step 2/4] > 经历萃取
  → 解析草稿，提取 3 段经历，结构化为 ExperienceItem 列表

[Step 3/4] > STAR 亮点打磨（JD 定制）
  → 结合 JD 关键词，用 STAR 原则重构每条经历
  → 关键词覆盖率: 85%

[Step 4/4] > 排版适配与导出
  → 填充 Word 模板
  → 输出: output/【管培生】姓名_定向简历.docx
```

### Step 6 — 展示输出

打开生成的 Word 简历，展示：
- STAR 格式化的经历描述
- JD 关键词精准覆盖
- 排版整洁的 Word 简历

**旁白**：

> 只需一段乱糟糟的经历草稿和目标岗位 JD，
> 智能体自动完成 JD 分析、经历萃取、STAR 改写、排版导出，
> 全程无需手动编辑，直接生成可投递的 Word 简历。

## 场景二：模拟面试练习（约 3 分钟）

**触发 Pipeline**：`interview_practice`  
**Pipeline 链路**：岗位能力建模 → 结构化出题 → 四维评分 → 报告生成

### Step 1 — 输入自然语言指令

在 AutoClaw 对话框中输入：

```
我想练习管培生岗位的模拟面试
```

### Step 2 — GLM 意图识别

GLM-4-Plus 识别意图，命中关键词「模拟面试」「管培生」，路由至 `interview_practice` Pipeline。

### Step 3 — 加载能力模型 & 出题

AutoClaw 从 `references/competency_models/guanpeisheng.json` 加载管培生能力模型，生成结构化面试题：

```
已加载管培生能力模型（5 维度: 领导力/沟通/分析/执行/学习力）
生成 3 道面试题：

Q1 [行为面 | 中等]: 请描述一个你在团队中发挥领导作用的经历
    考察维度: leadership

Q2 [技术面 | 中等]: 如果让你分析一个新市场的进入可行性，你会从哪些维度入手？
    考察维度: analytical_thinking

Q3 [综合面 | 较难]: 如果你是公司新业务的负责人，前90天你会做什么？
    考察维度: leadership
```

### Step 4 — 逐题作答

对每道题输入回答，例如：

**Q1 回答**：
```
大二在学生会文艺部，我负责策划迎新晚会。当时团队只有4个人，我主动分配了外联、宣传、节目筛选和后勤四个模块。外联方面我亲自跑了5家商家，最终拉到2000元赞助。晚会当天来了300多人，是当年参与人数最多的学生活动。
```

**Q2 回答**：
```
我会从市场规模、竞争格局、用户需求、盈利模型和风险评估五个维度入手。先做桌面调研了解行业趋势，再做用户访谈验证需求真伪，最后用数据建模估算可获取的市场份额。
```

**Q3 回答**：
```
前30天密集学习，走访每个部门了解业务流程和痛点。30-60天选定一个高杠杆切入点，制定方案并争取资源。60-90天推动第一个小闭环落地，用数据证明价值，同时建立跨部门协作机制。
```

### Step 5 — 四维评分结果

每道题回答后，AutoClaw 实时返回评分：

```
Q1 评估结果:
  内容完整度: 82/100
  逻辑条理性: 78/100
  STAR 结构性: 85/100
  关键词覆盖: 70/100
  综合得分: 79/100 (等级: B)
  改进建议: 补充量化结果（如晚会参与人数增长率），增加与管培生岗位的关联描述
```

### Step 6 — 生成面试报告

所有题目评估完毕后，AutoClaw 自动汇总：

```
正在生成面试练习报告...
→ 输出: output/面试练习报告_管培生.docx
```

打开报告，展示：
- 各题回答 + 评分详情
- 维度得分雷达图描述
- 薄弱项识别 + 改进建议

**旁白**：

> 面试题不是泛化的通用题，而是基于管培生岗位能力模型定制的。
> 四维评分体系从内容完整度、逻辑条理性、STAR 结构、关键词覆盖四个角度量化评估，
> 最终生成完整的练习报告，帮助用户针对性改进。

## 场景三：实习职位自动聚合（约 2 分钟）

**触发 Pipeline**：`internship_aggregator`  
**Pipeline 链路**：跨平台抓取 → 匹配评分 → Excel 对比表

### Step 1 — 输入自然语言指令

在 AutoClaw 对话框中输入：

```
帮我找上海的 AI 产品实习，生成对比表
```

### Step 2 — GLM 意图识别

GLM-4-Plus 识别意图，命中关键词「找实习」「上海」「对比表」，路由至 `internship_aggregator` Pipeline。

### Step 3 — 跨平台岗位抓取

```
[Step 1/3] > 跨平台岗位抓取
  → 平台: BOSS直聘, 实习僧, 牛客
  → 关键词: AI 产品实习 | 城市: 上海
  → 自动翻页抓取中...
  → 抓取完成: 共 25 个岗位
```

### Step 4 — 智能匹配评分

```
[Step 2/3] > 智能匹配评分
  → 四维评分: 技能契合度(40%) + 出勤匹配(20%) + 城市匹配(20%) + 学历匹配(20%)
  → 评分完成，Tier 分级: 强推荐 5 个 / 推荐 12 个 / 观望 8 个
```

### Step 5 — 生成 Excel 对比表

```
[Step 3/3] > 生成 Excel 对比表
  → 输出: output/上海_AI产品实习_实习对比表.xlsx
```

打开 Excel 文件，展示：
- 岗位名称、公司、薪资、匹配分数、Tier 等级
- 条件格式色块（绿色=强推荐，黄色=推荐，红色=观望）
- 冻结标题行，自适应列宽

**旁白**：

> 不用再逐个平台手动搜索了。
> 智能体自动跨平台抓取、智能匹配评分、生成一目了然的对比表，
> 帮助学生快速锁定最值得投递的岗位。

## 录制建议

1. **先录场景一**：最核心、最稳定，出错率最低
2. **再录场景二**：交互多，需要流畅作答，建议提前背好回答要点
3. **最后录场景三**：依赖外部网络，不确定性最大
4. **每个场景单独录制**，后期剪辑拼接
5. **日志区保持可见**，展示 Pipeline 执行过程是重要加分项
