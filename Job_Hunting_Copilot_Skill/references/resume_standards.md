# 简历内容与打磨规范 (Resume Standards)

> 本文件是智能体执行「经历萃取」与「亮点打磨」Skill 的核心参考标准，所有输出必须严格遵守。

---

## 1. 信息萃取数据结构 (JSON Schema)

每段提取的经历必须严格符合以下结构：

```json
{
  "time": "YYYY.MM-YYYY.MM",
  "organization": "所在公司、学校组织或比赛名称",
  "role": "担任的具体职位或角色",
  "raw_actions": ["原始动作描述1", "原始动作描述2"]
}
```

---

## 2. 核心方法论：STAR 原则

所有经历描述必须包含以下四个要素，缺一不可：

| 要素 | 含义 | 示例 |
|:---|:---|:---|
| **S (Situation) 情境** | 当时面临的背景或挑战 | 社团面临招新压力、项目缺乏资金 |
| **T (Task) 任务** | 你的具体目标或职责 | 需在 2 周内拉到 2000 元赞助 |
| **A (Action) 行动** | ⭐**核心**：你具体做了什么动作？用了什么工具/方法？ | 通过 3 轮商务洽谈，设计赞助权益方案 |
| **R (Result) 结果** | 量化的成果或可衡量的影响 | 最终到账资金 2500 元，超额完成 25% |

---

## 3. 职场化词汇对照表 (Impact Verbs)

禁止使用口语/模糊动词，必须使用具备"影响力"的行动动词：

| 原始描述（口语化/非专业） | ✅ 推荐转换（职业化） | 体现的核心素质 |
|:---|:---|:---|
| 参与了... | **主导 / 统筹 / 独立负责** | 领导力与主动性 |
| 学会了... | **熟练运用 / 系统掌握** | 专业技能与学习力 |
| 写了文章/推文 | **撰写并发布 / 内容创作与分发** | 传播能力 |
| 帮老师/同学干活 | **协助推进 / 支撑执行** | 协作与执行力 |
| 拉到了赞助 | **独立开展商务拓展（BD） / 洽谈资金支持** | 沟通与商业变现能力 |
| 办了活动 | **统筹策划并落地活动全流程** | 项目管理能力 |
| 整理表格 | **数据清洗、整理与分析** | 数据处理能力 |
| 建了群/维护群 | **搭建并精细化运营用户社群** | 用户运营能力 |

---

## 4. 校招 / 管培生专项优化要点

针对零经验毕业生，重点挖掘以下**隐性软素质**，而非单纯罗列任务：

1. **学习力 (Learning Agility)**：在短期内自学新工具、新领域并产出结果的经历。
2. **抗压性 (Resilience)**：在资源有限、时间紧迫或高强度环境下完成任务的经历。
3. **协作力 (Collaboration)**：跨学院、跨社团、跨专业调动多方资源的经历。
4. **领导力 (Leadership)**：带领他人、分配任务、推动项目落地的经历（哪怕规模小）。

## 5. 影响力量化公式

### 常见量化维度

| 场景类型 | 量化指标示例 |
|:---|:---|
| 效率提升 | 提升效率 X%、节省工时 Y 小时/天 |
| 收入/资金 | 带来收入/资金 Z 元，超额完成 X% |
| 用户/覆盖 | 触达 X 名用户，社群规模 Y 人 |
| 传播影响 | 阅读量 X，转发 Y 次，涨粉 Z |
| 团队规模 | 统筹 X 人协作，管理 Y 人跟进 |

### 缺乏数据时的占位符规范
- 使用 `[X]` 标记所有需要补充的数字
- 示例：`提升了 [X]% 的用户留存率，影响 [X] 名核心用户`

---

## 6. ATS 关键词优化规范

### 关键词类型
- **硬技能**：编程语言、工具软件、专业证书（如 Python, SQL, Axure, PMP）
- **软技能**：沟通协作、项目管理、数据驱动决策
- **职责关键词**：从 JD 中提取的核心动词和名词短语

### 覆盖率评估标准
| 覆盖率 | 状态 | 建议 |
|:---|:---|:---|
| ≥ 80% | 🟢 高 | 通过 ATS 概率高 |
| 50%~79% | 🟡 中 | 需针对性补充关键词 |
| < 50% | 🔴 低 | 建议重写核心经历描述 |

---

## 7. 简历自检清单 (Final Checklist)

提交前必须逐项确认：

- [ ] 每条描述都以**行动动词**开头（无"我"字开头的句子）
- [ ] 每段经历至少含 **2 个量化数字**（或已注明 `[X]` 待填）
- [ ] 关键词覆盖目标 JD 的核心要求（覆盖率 ≥ 70%）
- [ ] 无口语化、无主观情绪词（"辛苦地"、"努力地"）
- [ ] 无空泛的"自我评价"段落
- [ ] 每条经历描述控制在 **1~2 行**以内
- [ ] 时间格式统一：`YYYY.MM - YYYY.MM`
