# 岗位评估与评分规范

## 1. 匹配度评分维度 (总分 100 分)

| 维度 | 权重 | 评估要点 |
|-----|-----|---------|
| 技能契合度 | 40分 | JD硬技能要求与用户技能库的重合比例 |
| 学历与年级匹配 | 20分 | 学历、年级是否符合岗位的转正/实习要求 |
| 实习时长与到岗 | 20分 | 每周出勤天数及连续实习月数是否满足JD要求 |
| 城市与通勤 | 20分 | 目标城市是否一致，异地可行性 |

## 2. 匹配理由输出规范

| 分数段 | 语气 | 内容要求 |
|-------|-----|---------|
| 高分 (80-100) | 肯定 | 指出1-2项最匹配的核心技能/经历 |
| 中分 (60-79) | 客观 | 指出符合项 + 缺失的关键技能 |
| 低分 (<60) | 直接 | 明确不匹配的硬性条件 |

### 示例
- **高分**：极度匹配：你的 Python 技能和上一段大厂数据分析经历完美契合该岗位需求。
- **中分**：勉强匹配：符合基本要求，但 JD 强调的 B 端产品经验你较为欠缺。
- **低分**：不建议投递：该岗位明确要求每周全职 5 天，且需计算机科班出身。

## 3. 战略级岗位分析维度

### 岗位核心定位
一句话概括该岗位在团队/公司中的核心价值

### 关键职责权重排序
列出3-5项核心职责，按重要性推测排序

### 隐藏要求识别
从JD字面描述推断的潜在要求：
- "快速成长" → 可能意味着变化快、需抗压
- "自驱力强" → 可能缺乏完善培训体系
- "扁平化管理" → 可能需独立解决问题

### 大坑风险警示
| 风险类型 | 预警信号 |
|---------|---------|
| 行业下行 | 公司/行业裁员、融资困难 |
| 岗位边缘 | JD描述模糊、汇报层级不清 |
| 压力过大 | "能接受加班"、"高强度"、"快节奏" |
| 发展受限 | 纯执行岗、无晋升路径描述 |

## 4. 投递优先级评估

### Tier划分标准
| 优先级 | 条件 |
|-------|-----|
| Tier 1 (主攻) | 匹配度≥75% + 推荐指数高 + 无明显大坑 |
| Tier 2 (次选) | 匹配度50-74% 或 有1个可控风险 |
| Tier 3 (观望) | 匹配度<50% 或 存在多个风险点 |

### 差异化准备策略
- **Tier 1**：深度研究公司业务，准备定制化简历和作品集
- **Tier 2**：针对性补强短板，准备解释差距的话术
- **Tier 3**：快速投递，作为练手机会
